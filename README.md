# 羽田空港P3駐車場 空き通知(GitHub Actions + ntfy.sh)

7/26・7/27のP3(第2ターミナル)駐車場に空きが出たら、ntfy.sh経由でスマホに通知します。
サイトが内部で使っているJSON API(/airport2/app/calendar)を直接呼び出す方式なので、
ブラウザ操作(Playwright)は不要です。

## セットアップ手順

### 1. ntfyアプリをインストールして購読する
1. スマホに ntfy アプリをインストール(iOS App Store / Google Play)
2. アプリ内で「+」から新しいトピックを購読する(推測されにくいランダムな名前推奨)

### 2. このリポジトリをGitHubにpushする(またはファイルを直接編集)

### 3. GitHub Secretsを設定
リポジトリの Settings → Secrets and variables → Actions で以下を追加:
- `NTFY_TOPIC` … 手順1で決めたトピック名

### 4. 動作確認
「Actions」タブ→該当ワークフロー→「Run workflow」で手動実行し、ログを確認する。

## 前回の状態の保存について
GitHub Actionsは実行のたびに環境がまっさらになるため、`actions/cache`を使って
`state.json`(前回チェック時に空きがあった日付)をワークフロー間で引き継いでいる。
これにより「状態が変化した時だけ」通知が送られ、同じ状態のまま毎回通知が来る
(ntfy側のレート制限にかかる)ことを防いでいる。

## 注意事項
- cronは5分おきに設定していますが、GitHub側の仕様として混雑時は指定時刻通りに
  実行されず遅延することがあります(公式にそう案内されています)。
- ntfy.sh(無料の公開サービス)は送信元IPアドレスごとにレート制限があります。
  GitHub ActionsやGoogle Apps Scriptは共有IPアドレスを使うため、自分と無関係な
  他ユーザーのトラフィックによって429エラー(レート制限)が出ることがあります。
  頻発する場合は、送信元IPに依存しないDiscord Webhookなどへの切り替えを検討する。
- このサイトの利用規約・アクセス監視に関する記載を必ず確認し、
  自動アクセスが許容される範囲かご自身でご確認ください。
